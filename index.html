AI Agent에게 코드 생성을 맡기는 방법

샘플코드를 AI에게 가이드

"example.org.sample 패키지의 SampleApi.java를 참고해서 TutorApi.java를 작성해줘"

장점

  • 별도 설정 없음 (단순함)
  • 가장 근접한 코드 작성

단점

  • 문서 작성 포맷 한계
  • 샘플 코드 유지 필요
  • 여러 케이스의 샘플 코드
  • 샘플 파일 종류/항목 기억

Confluence MCP

"https://confluence.hmg-corp.io/pages/viewpage.action?pageId=1286890412 를 참고해서 TutorApi.java를 작성해줘"

장점

  • 공유 가능한 형태
  • 문서 작성 한계 없음

단점

  • 문서 종류/항목 기억
  • 검색이 어려움 (cql..)
  • 연결 시 레이턴시

Lago MCP

"lago MCP로 코드 표준 문서 중 API 레이어를 확인하고, TutorApi.java를 작성해줘"

장점

  • 공유가 가능한 형태
  • 원하는 경로의 파일 관리
  • 문서 작성 한계 없음
  • file 경로 사용 시 레이턴시 거의 없음
  • 검색이 용이함 (BM25)

단점

  • 대량 문서를 다루기 어려움

문서 관리 MCP 소개 (Lago MCP)

해달:라고(haedal:lago) : Model Context Protocol(MCP) 기반의 문서 검색 및 관리 시스템

프로젝트 요소

  • 1. MCP 서버: Claude와 같은 AI 어시스턴트가 사내 문서에 직접 접근할 수 있는 표준 프로토콜
  • 2. BM25 기반 검색: 키워드 매칭과 의미 기반 검색을 통한 정확한 문서 검색
  • 3. Docsify 웹 서버: 브라우저에서도 접근 가능한 문서 포털 제공
  • 4. 동적 문서 경로: 중앙집중형 문서 조회 뿐만 아닌, 개인 폴더, URL을 통한 경로 지정을 통한 문서 검색
  • 5. llms.txt: llm과 인간이 모두 읽을 수 있고, 정규식처리나 파서를 통한 처리가 편리한 형태의 규약
  • 6. markdown: 간단한 문법으로 텍스트를 작성하여 HTML로 변환할 수 있는 경량 마크업 언어

해결하고 싶은 문제

  • 1. 반복적인 업무 대응: 같은 질문에 대해 담당자가 반복적으로 답변
  • 2. 분산된 업무 매뉴얼: 여러 시스템에 흩어져 있는 문서들로 인한 정보 접근성 저하
  • 3. 비효율적인 온보딩: 신입/경력 사원이 필요한 정보를 찾기 위해 소요되는 시간
  • 4. 담당자 찾기의 어려움: "누가 이걸 담당하지?" 형태의 반복적인 문의

Lago Mcp 활용

  • 1. AI 기반 자동 응답: 반복적인 질문을 AI가 대신 처리
  • 2. 통합 문서 검색: 팀/프로젝트 문서를 하나의 인터페이스에서 검색
  • 3. 즉시 접근 가능한 정보: AI Agent 채팅창에서 바로 업무 정보 조회
  • 4. 담당자 정보 자동 제공: 시스템별 담당자 정보를 문서와 함께 제공

코드 생성 외 활용 방법

AI Agent와 MCP를 활용한 다양한 업무 자동화 및 효율성 개선 사례

문서 자동 번역 및 요약

다국어 문서 처리

기술 문서, 매뉴얼, 사양서 등을 자동으로 번역하고 핵심 내용을 요약하여 팀 간 커뮤니케이션을 원활하게 지원합니다.

코드 리뷰 자동화

품질 관리 강화

코딩 표준 준수, 보안 취약점 검사, 성능 최적화 제안 등을 자동으로 수행하여 코드 품질을 향상시킵니다.

업무 프로세스 문서화

지식 관리 체계화

회의록, 결정사항, 업무 절차 등을 자동으로 구조화하고 검색 가능한 형태로 문서화하여 조직 지식을 체계적으로 관리합니다.

증거는 분명합니다: AI를 받아들이거나, 아니면 이 직업을 떠나세요.

AI를 워크플로우에 깊이 통합하고 있는 22명의 개발자를 대상으로 한 최신 현장 연구에서 놀라운 경향이 드러났습니다: 초기 회의론을 극복한 개발자들은 극적으로 높은 야심, 기술적 유창함, 그리고 직업 만족도를 보여줍니다. 그들은 더 적은 코드를 작성하는 것이 아니라 오케스트레이션을 통해 더 복잡하고 시스템 레벨의 작업을 가능하게 하고 있습니다. 이는 교육자들에게도 마찬가지로 적용됩니다.

이러한 변화는 가설이 아닙니다. 지금 일어나고 있는 현실입니다. 개발자들은 명확한 도입 단계를 거칩니다 - 회의적인 호기심에서 전략적 AI 협력자로 - 그리고 최종 단계에 도달한 개발자들은 개발자로서의 정체성이 변화했다고 말합니다. 그들의 초점은 더 이상 코드 생산이 아니라 시스템 설계, 에이전트 지시, 그리고 결과물 검증에 있습니다. "제 다음 직책은 코드의 크리에이티브 디렉터가 될 수도 있습니다"라고 한 개발자가 말했습니다. 이는 과장이 아니라 현실입니다.

우리가 보고 있는 것들:

  • AI는 향후 2-5년 내에 90%의 코드를 작성할 예정입니다.
  • 개발자들은 걱정하지 않습니다. 그들은 앞으로 다가올 변화에 대해 낙관적이고 현실적입니다.
  • 새로운 기술이 중요해졌습니다. 예: 에이전트 오케스트레이션, 반복적 협업, 그리고 비판적 검증.
  • 시간 절약? 물론입니다. 하지만 진짜 변화는 야심입니다. 개발자들은 비용을 낮추는 것이 아니라 한계를 높이고 있습니다.

이는 교육에도 엄청난 영향을 미칩니다. 문법만 가르치는 것은 이제 구식입니다. 학생들은 이제 AI를 가이드하고, 그 작업을 비판하며, 학문을 넘나드는 사고를 배워야 합니다. 평가는 AI와의 협업을 측정해야 하며, AI로부터 격리된 상태가 아니어야 합니다.

이는 더 이상 생산성의 문제가 아닙니다. 재창조의 문제입니다. 소프트웨어 개발자의 직업은 사라지지 않습니다. 다시 태어나고 있습니다.
